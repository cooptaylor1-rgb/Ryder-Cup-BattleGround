## Review Process

1. **Draft** → Author creates or updates document
2. **Reviewed** → Peer review completed, feedback addressed
3. **Final** → Approved by owner, ready for reference

### Review Cadence Definitions

- **Per Release**: Review before each version release
- **Monthly**: Review at start of each month
- **Quarterly**: Review at start of each quarter
- **Annual**: Review at start of each year
- **As Needed**: Review when significant changes occur
